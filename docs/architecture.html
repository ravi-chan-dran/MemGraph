üéØ System Overview

MemoryGraph is a production-ready AI system that provides persistent memory capabilities to Large Language Models (LLMs). The system enables AI to learn, remember, and reason about information across multiple interactions, creating a more human-like AI experience.

Core Principles

  • Persistent Memory: AI retains information across sessions
  • Multi-Modal Storage: Facts, episodes, and graph relationships
  • Transparent Reasoning: AI shows how it reaches conclusions
  • Selective Forgetting: Users control what AI remembers
  • Real-Time Learning: AI learns from new interactions immediately

üíæ Storage Architecture

SQLite (Facts)

  • ACID compliance
  • Fast lookups
  • Embedded database
  • Structured facts storage

ChromaDB (Episodes)

  • Vector similarity search
  • Persistent storage
  • Metadata filtering
  • Semantic search

Neo4j (Graph)

  • Graph queries
  • Relationship traversal
  • Constraint enforcement
  • Path finding

SQLite Schema

CREATE TABLE facts (
    guid TEXT,
    key TEXT,
    value TEXT,
    confidence REAL,
    source TEXT,
    ts TEXT,
    PRIMARY KEY(guid, key)
);

ChromaDB Collection

Collection: "episodes_mem"
Metadata: {
    "guid": str,
    "timestamp": str,
    "channel": str,
    "importance": float,
    "tags": list
}

Neo4j Graph Structure

(:User {guid: "plan_sponsor_acme"})
(:Fact {key: "match_formula", value: "100% of first 3%", confidence: 1.0})
(:Entity {name: "retirement_plan", type: "Policy"})
(:User)-[:HAS_FACT]->(:Fact)
(:Entity)-[:RELATED_TO]->(:Fact)

üï∏Ô∏è Graph Database Design

Node Types

User: {guid, created_at}
Fact: {key, value, confidence, source, ts}
Entity: {name, type, aliases}
Episode: {summary, importance, tags}

Relationship Types

HAS_FACT: User ‚Üí Fact
HAS_EPISODE: User ‚Üí Episode
RELATED_TO: Entity ‚Üí Entity
MENTIONS: Episode ‚Üí Entity
HAS_FORMULA: Entity ‚Üí Fact
SCHEDULED_FOR: Event ‚Üí Date

Constraints

CREATE CONSTRAINT user_guid_unique FOR (u:User) REQUIRE u.guid IS UNIQUE;
CREATE CONSTRAINT entity_key_unique FOR (e:Entity) REQUIRE (e.name, e.type) IS UNIQUE;
CREATE CONSTRAINT fact_key_unique FOR (f:Fact) REQUIRE (f.key, f.guid) IS UNIQUE;

üîí Security & Privacy

Authentication

  • MCP Server: Bearer token
  • API Endpoints: No auth (demo)
  • Production: JWT/OAuth2

Data Privacy

  • Selective forgetting
  • Data isolation by GUID
  • Confidence filtering
  • Source tracking

Data Retention

  • Facts: Persistent until deleted
  • Episodes: Can be redacted
  • Graph: Soft-delete relationships
